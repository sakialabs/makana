# Documentation

This folder contains all planning, design, and reference documents for Makana.

## Files

**Current Status & Next Steps**
- `tasks.md` - Current action items and recommendations for next focus
- `CHANGELOG.md` - Complete implementation history and version tracking
- `roadmap.md` - Product roadmap with v0 (complete), v1, and v2 plans

**Design & Architecture**
- `design.md` - System architecture, data models, and correctness properties
- `testing.md` - Testing strategy and guidelines
- `deployment.md` - Deployment instructions and configuration

**Vision & Philosophy**
- `vision.md` - Product vision and core principles
- `manifesto.md` - Beliefs about practice, energy, and intentional strength
- `tone.md` - Voice and language guidelines for all communication

**Visual Design**
- `visuals.md` - Design system, color palette, typography, and UI principles

**Feature Planning**
- `gamification.md` - Practice Forms and recognition (v1)
- `onboarding.md` - Onboarding flow design

## Reading Order

**New to Makana?** Start here:
1. `vision.md` - Understand what we're building and why
2. `manifesto.md` - Core beliefs that guide all decisions
3. `roadmap.md` - Product scope and implementation status
4. `design.md` - How the system works
5. `tasks.md` - What to work on next

**Ready to contribute?**
1. `tasks.md` - See current priorities
2. `CHANGELOG.md` - Review what's been completed
3. `../CONTRIBUTING.md` - Development workflow and guidelines

## Principles

All documentation follows Makana's tone: calm, direct, respectful. No pressure, no urgency, no blame. We value clarity over cleverness, stability over novelty.
